Goal of Project

To create a puzzle game inspired by Plock, but with improved game play. We take the fundamental structure of our game from Plock but modify the game so that it rewards players who think over those who mindlessly click.

How to Play: The basic goal of the game is to match similar colored blocks. As the blocks are cleared new blocks fill in from the top of the board. As you score points, your bomb bar grows and when it is full a bomb is placed on the board in the form of a star. When you click on the bomb, an area around it is cleared out and any new blocks cleared will be worth even more points.

Our Design document explains how we:
  • Re-created and improved upon the fundamental game dynamics of Plock
  • Created a customizable, versatile user interface including high scores and settings
  • Designed 3 distinct game modes
  • Incorporated user feedback to alter the scoring and shuffling mechanisms to allow for more exciting game play

Guide to Setting Up Project

    The following describes a tested set of steps to create a workable project. The project has been tested on APIs 10, 15, and 16.

  1. Download files from the BOMBSTAR.zip file at the following link. Link to Dropbox folder.
  2. Create a Qt project named BOMBSTAR with the following properties:
    • Arm v5
    • Android NDK Tool chain = 4.4.3
    • Ant v1.8.2
  3. Delete default mainwindow.cpp and mainwindow.h. Failure to do so will result in a failure to compile.
  4. Copy C++ files to the project directory.
  5. Create or use an AVD with the following or similar specifications:
    • API 16
    • SD Card Size = 1000M
    • Snapshot: Enabled
    • Skin: Built-in = WVGA800
    • Abstracted LCD Density = 240
    • Max VM Application Heap Size = 48
    • Device RAM size = 4096
  6. Be sure the following files are included in the project directory:
    • Block.h / Block.cpp
    • Colors.h / Colors.h
    • highScores.h / highScores.cpp
    • mainWindow_UI.h / mainWindow_UI.cpp
    • Score.h / Score.cpp
    • ScoreFrame.h / ScoreFrame.cpp
    • main.cpp
  7. Build project.
  8. Run project.
  9. Play game as described in the Instructions tab in this reference manual.
  10. Read the Design document to understand how the game is created and why it's created that way.
  11. Refer to the Doxygen Appendix for a quick reference to the objects used in the code. Note that the Doxygen page is self-contained and does not link back to this main page.
